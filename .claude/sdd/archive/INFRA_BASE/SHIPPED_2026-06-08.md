## Lessons Learned

### Processo

- **SDD funciona bem para features de infra**: O fluxo brainstorm → define → design → build produziu um documento de design com 8 code patterns copy-paste ready que foram implementados sem ambiguidade. Para features mais simples como INFRA_BASE, é possível compactar brainstorm e define na mesma sessão.
- **Perguntas one-at-a-time no brainstorm aceleram o alinhamento**: Fazer 5 perguntas focadas (gerenciador, Docker, layout, LLM, amostras) foi mais eficiente do que um formulário de requisitos. A quinta pergunta (schemas Gold) trouxe informações que melhoraram o design (`fundo_mensal` já tem `alpha_selic` pré-calculado).

### Técnico

- **`uv` instala `[dependency-groups]` por padrão**: Diferente de `pip install -e .`, o `uv sync` inclui todos os grupos de dependências. Em CI/CD de produção, usar `uv sync --no-dev` para excluir o grupo dev. Documentado como decisão autônoma no BUILD_REPORT.
- **`hatchling` exige que `readme` referenciado exista**: `readme = "README.md"` no `pyproject.toml` causa falha de build se o arquivo não existe. Solução: remover a chave ou criar o arquivo. Optou-se por remover (README não era escopo da INFRA_BASE).
- **`ANN101`/`ANN102` foram removidos do ruff**: Regras de annotação de `self` e `cls` foram extintas na versão 0.15+. Remover do `ignore` elimina o warning e mantém a config limpa.
- **`lru_cache` em `get_settings()` é essencial para testabilidade**: O cache garante que `.env` seja lido uma vez por processo, mas a função `get_settings.cache_clear()` permite que os testes criem novos `Settings()` isolados via `monkeypatch`.

### Comunicação

- **Schemas Gold fornecidos no brainstorm ancoraram o design**: O usuário forneceu os 4 schemas no passo de amostras. Isso revelou que `gold_cvm.fundo_mensal` já tem `alpha_selic` e `alpha_ipca` calculados — o agente não precisará fazer joins cross-domain, simplificando a feature SQL_TOOL.
- **Decisão LLM clara desde o brainstorm evitou ambiguidade no build**: Claude `claude-sonnet-4-6` configurável via `.env` foi decidido antes do DEFINE. O `config.py` implementou `model_name` com default exato sem precisar de perguntas adicionais.

### Ferramentas e Bibliotecas

- **`pydantic-settings`**: Validação automática, mensagens de erro claras, carregamento de `.env` nativo e `model_config` declarativo fazem dela a escolha certa para configuração em projetos LLM. O padrão `Settings(BaseSettings)` é amplamente usado no ecossistema LangChain/FastAPI.
- **`uv`**: 182 pacotes resolvidos em 4ms após o primeiro lock. Comparado ao `pip install`, a diferença de velocidade é substancial. O `uv.lock` garante reprodutibilidade entre ambientes sem configuração adicional.
- **Chainlit**: O decorator `@cl.on_chat_start` + `@cl.on_message` é suficiente para um entry point funcional. A integração com `pydantic-settings` via `get_settings()` no escopo do módulo é simples e eficiente.
