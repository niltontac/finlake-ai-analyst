## Summary

A feature PROMPTS entrega os dois `ChatPromptTemplate` que são o cérebro do agente Text-to-SQL do FinLake Brasil. `get_sql_prompt()` instrui o Claude a gerar SQL PostgreSQL puro a partir de perguntas em português, com dois exemplos few-shot validados (P1: ranking de fundos por alpha_selic; P3: evolução da SELIC) e seis regras de domínio embutidas — incluindo a instrução crítica de retornar apenas SQL puro sem blocos markdown. `get_interpretation_prompt()` transforma o resultado da query em análise financeira contextualizada em português brasileiro, com referências a SELIC, CDI e alpha como benchmarks de mercado. Os dois templates estão prontos para consumo pelo `AGENT_CORE` via `from finlake_analyst.prompts import get_sql_prompt, get_interpretation_prompt`.

## Lessons Learned

### Processo

- **Brainstorm com grounding direto no SQL_TOOL acelerou tudo.** As 5 queries validadas do Brainstorm anterior foram o material exato de few-shot. Não houve nenhuma ambiguidade sobre quais exemplos usar — já estavam validados contra o banco real. O investimento em grounding em fases anteriores paga dividendos nas fases seguintes.
- **O DESIGN com code patterns copy-paste tornou o build trivial.** Todo o conteúdo dos arquivos estava especificado no DESIGN — o build foi essencialmente transcrever patterns já validados. Nenhuma decisão de implementação foi necessária.

### Técnico

- **`\` dentro de strings triple-quoted suprime o newline seguinte** — padrão correto para quebrar linhas longas em strings Python sem alterar o conteúdo. Útil para respeitar `line-length=100` do ruff em strings de prompt sem usar concatenação de strings (que obscurece o conteúdo).
- **LangChain detecta `{variable}` automaticamente em `ChatPromptTemplate.from_messages()`** — qualquer `{...}` na string é tratado como variável. As strings de few-shot SQL não continham `{...}`, então não houve falso-positivo de variável. Importante verificar isso antes de adicionar exemplos com JSON ou outros `{...}` no prompt.
- **Testes puramente estruturais (sem LLM) são rápidos e suficientes para prompt templates.** Os 8 testes rodam em ~0.03s e verificam tudo que pode ser verificado estaticamente: variáveis, presença de conteúdo crítico. A qualidade do output do LLM é validada manualmente no AGENT_CORE.
- **`ADR-002` (variável `{schema}` vs parâmetro)** — a separação "template com variáveis" vs "função que pré-injeta" é importante para observabilidade. O LangFuse vai mostrar `schema` e `question` como inputs separados no trace, facilitando debugging.

### Comunicação

- **A regra "SQL puro sem markdown" foi adicionada pelo usuário no Brainstorm** — sem esse reforço, o Claude tem forte tendência de envolver SQL em ` ```sql``` `. Essa instrução precisou ser explícita e está diretamente testada em `test_sql_prompt_prohibits_markdown_blocks`.
- **Escopo minimal foi chave.** Nenhum prompt de erro, clarificação ou memória foi incluído — todos delegados ao AGENT_CORE. Manter os prompts com responsabilidade única torna o sistema mais previsível.

### Tools & Libraries

- **`langchain_core.prompts.ChatPromptTemplate`** — zero dependências além do que já estava no projeto (`langchain-core` é dependência transitiva de `langchain`). A escolha de `langchain_core` (não `langchain`) é mais estável e não gera o `DeprecationWarning` que `langchain-community` gera.
- **`ChatPromptTemplate.partial()`** — o método `.partial(schema=schema_str)` do AGENT_CORE vai pré-vincular o schema uma vez por sessão, deixando apenas `{question}` como variável livre. Isso é mais eficiente do que reinjetar o schema em cada mensagem.

## Recommendations for Future Work

| Área | Recomendação |
|---|---|
| `AGENT_CORE` | Chamar `get_sql_prompt().partial(schema=schema_str)` no `on_chat_start`; `get_interpretation_prompt()` pode ser instanciado fresh por request (é stateless) |
| Qualidade dos prompts | Após o AGENT_CORE funcionar, usar LangFuse para avaliar quais perguntas geram SQL incorreto — iterar no few-shot com exemplos do domínio real |
| Few-shot expansion | Se alpha_selic queries persistirem com erros, adicionar P4 (PL médio por tipo) e P5 (captação positiva com SELIC > 10%) como exemplos adicionais |
| Prompt versioning | Com LangFuse, configurar `prompt_name` e `prompt_version` para rastrear qual versão do prompt gerou cada resultado — útil para A/B de prompts |
