## Summary

A feature AGENT_CORE entrega o orquestrador central do FinLake Analyst — o `StateGraph` LangGraph que conecta todos os blocos já entregues em um agente Text-to-SQL funcional. O `app.py` Chainlit foi atualizado do placeholder para a implementação real: o usuário digita uma pergunta financeira e recebe uma análise em português em tempo real via streaming.

**Fluxo:** pergunta → `generate_sql` → `execute_sql` → (retry loop com max 2 falhas) → `interpret_result` → streaming ao usuário. Caminho alternativo: `execute_sql` com 2 falhas → `handle_error` → mensagem em português sem expor detalhes técnicos.

**Padrão de implementação:** factory functions (closures) para injeção de dependência nos nós, permitindo testes unitários com `AsyncMock` sem necessidade de `@patch`. O routing é uma função pura `_route_after_execute()` testável em isolamento.

## Lessons Learned

### Processo

- **O DESIGN com code patterns copy-paste tornou o Build linear.** Todos os 7 arquivos foram escritos diretamente a partir dos patterns do DESIGN sem nenhuma decisão de implementação necessária. Zero tempo gasto em "qual é o jeito certo?" — o DESIGN já respondeu. O investimento em um DESIGN detalhado (com assinaturas exatas, casos de erro, e patterns completos) pagou dividendos no Build.

- **Separar routing em função pura foi a decisão mais valiosa do DESIGN.** `_route_after_execute()` é uma função síncrona sem dependências externas — testada em 6 casos independentes sem nenhum mock. Se o routing estivesse embutido no grafo como lambda inline, seria impossível de testar em isolamento. A disciplina de extrair lógica para funções puras é especialmente valiosa em grafos LangGraph.

### Técnico

- **Factory functions (closures) como nós LangGraph.** O padrão `make_X_node(deps) -> async def node(state) -> dict` é a forma mais limpa de injetar dependências em nós LangGraph sem modificar a assinatura exigida pelo framework. Em testes, instanciar a factory diretamente com `AsyncMock` é mais simples do que usar `@patch` no módulo onde o LLM é criado. Esse padrão é recomendado para qualquer novo grafo neste projeto.

- **`asyncio_mode = "auto"` elimina `@pytest.mark.asyncio` de todos os testes.** Já estava configurado em `pyproject.toml` — nenhum decorator adicional necessário. Todos os `async def test_...` funções rodam automaticamente. Reduz boilerplate consideravelmente.

- **`retry_count` incrementado em `execute_sql` (não em `generate_sql`).** Incrementar onde a falha é detectada (execute_sql) em vez de onde a próxima tentativa começa (generate_sql) torna a lógica causal e o routing previsível: `_route_after_execute` lê o estado já atualizado. Se fosse o contrário, o routing precisaria de lógica adicional para distinguir "falhou e vai tentar" de "falhou e chegou no limite".

- **`handle_error` sync em vez de async.** LangGraph aceita tanto nós sync quanto async. `handle_error` não faz I/O, então sync é a escolha correta. Isso também simplifica os testes (chamada direta sem `await`).

- **Captura de `handle_error` output via `on_chain_end` no `app.py`.** A abordagem de capturar o output do nó `handle_error` via evento `on_chain_end` (em vez de rodar o grafo duas vezes) funciona no LangGraph 1.2.4 mas ainda precisa de validação manual com o Chainlit rodando. A-001 e A-002 (streaming via `astream_events`) são as únicas assumptions não validadas automaticamente.

### Ferramentas e Bibliotecas

- **LangGraph 1.2.4 instalado** (apesar de `pyproject.toml` especificar `>=0.2`). `CompiledStateGraph` em `langgraph.graph.state` e `END/START/StateGraph` em `langgraph.graph` funcionam conforme esperado nesta versão.

- **`ChatAnthropic(api_key=str)` funciona sem cast para `SecretStr`.** Pydantic v2 coerce `str` para `SecretStr` automaticamente no modelo. Nenhum `# type: ignore` necessário.
